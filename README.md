# Адміністратор картинг-клуба 🏎️

**Курсова робота з дисципліни «Об'єктно-орієнтоване програмування»**

Програма мовою C# для автоматизації роботи адміністратора картинг-клубу. Проєкт реалізує класичні принципи ООП, забезпечує надійне зберігання даних та інтуїтивне управління інформацією про клієнтів (гонщиків) і спортивні події (заїзди).

## ⚙️ Основний функціонал

* **Управління гонщиками (CRUD):** Реєстрація нових клієнтів, редагування профілів, безпечне видалення (із захистом від видалення учасників запланованих заїздів).
* **Управління заїздами (CRUD):** Створення заїздів, реєстрація учасників, фіксація часу кола (результатів).
* **Розумна статистика:** Автоматичне оновлення кількості завершених перегонів у профілі гонщика після зміни статусу заїзду.
* **Пошук та фільтрація:** Швидкий пошук інформації за визначеними критеріями.
* **Збереження даних:** Серіалізація та десеріалізація стану системи у форматі JSON.
* **Стійкість (Валідація):** Сувора перевірка вхідних даних (вік, формат телефону, унікальність ліцензії).

## 🏗️ Архітектура

Проєкт побудований на використанні централізованого класу-менеджера, який управляє колекціями двох взаємопов'язаних моделей даних:

1. **Клас `Racer` (Гонщик):** Модель даних клієнта (ID, ПІБ, вік, категорія, ліцензія, телефон, статистика).
2. **Клас `Session` (Заїзд):** Модель події на трасі (ID, дата, траса, тривалість, статус, результати учасників).
3. **Клас `KartingClubManager`:** Фасад-контролер. Інкапсулює бізнес-логіку, обробляє колекції, забезпечує цілісність даних (не копіює об'єкти, а пов'язує їх через унікальні ID) та взаємодіє з файловою системою.
