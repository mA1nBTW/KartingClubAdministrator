# 🏎️ Karting Club Manager

**Курсова робота з дисципліни «Об'єктно-орієнтоване програмування»**  
ХНУРЕ · ПЗПІ-25-3 · Павленко Михайло · 2026

Програма мовою C# для автоматизації роботи адміністратора картинг-клубу. Замінює ручний облік надійним цифровим інструментом: реєстрація гонщиків, формування заїздів, автоматичний підрахунок результатів і статистики.

---

## ⚙️ Функціонал

### Управління гонщиками
- Реєстрація нових клієнтів з перевіркою унікальності ліцензії
- Редагування профілю гонщика
- Безпечне видалення — система блокує видалення учасника запланованого заїзду
- Пошук та фільтрація за прізвищем або номером ліцензії в реальному часі

### Управління заїздами
- Створення заїзду з вибором учасників, траси, дати та тривалості
- Фіксація часу кола у зручному форматі (`58.34` / `1:23.45` / `1.23.45`)
- Автоматичне визначення переможця та підсвічування результату
- Перегляд завершених заїздів у режимі «тільки для читання»

### Статистика
- Автоматичне оновлення профілю після кожного заїзду: кількість заїздів, перемог, найкращий час кола
- Персональна статистика зберігається між сесіями

### Збереження даних
- Серіалізація / десеріалізація колекцій у JSON (`racers.json`, `sessions.json`)
- Програма коректно запускається без файлів — стартує з порожніми колекціями

## 🏗️ Архітектура

Проєкт побудований на принципі **розділення відповідальності (Separation of Concerns)** і включає три рівні:

```
UI Layer (Windows Forms)
    ↓
Logic Layer — KartingClubManager (Facade)
    ↓
Data Models — Racer, RacerStatistics, RaceSession
```

### Класи та зв'язки

| Клас | Роль | Зв'язок |
|------|------|---------|
| `Racer` | Профіль гонщика | — |
| `RacerStatistics` | Спортивні досягнення | **Композиція** з `Racer` |
| `RaceSession` | Спортивна подія | **Агрегація** гонщиків через `Dictionary<Guid, TimeSpan?>` |
| `KartingClubManager` | Фасад, бізнес-логіка | Агрегує `List<Racer>` та `List<RaceSession>` |

### Патерн Фасад

`KartingClubManager` — єдина точка взаємодії між UI і даними. Форми не знають про структуру колекцій і не звертаються до JSON напряму — лише через методи менеджера.

## 🖥️ Інтерфейс

| Форма | Призначення |
|-------|-------------|
| `MainForm` | Головне вікно: таблиці гонщиків і заїздів, пошук |
| `AddRacerForm` | Реєстрація нового гонщика |
| `EditRacerForm` | Редагування профілю |
| `CreateSessionForm` | Формування нового заїзду |
| `FinishSessionForm` | Введення результатів / перегляд заїзду |

**Клавіатурна навігація:** `F1` — довідка, `Enter` — підтвердження, `Esc` — скасування, `Delete` — видалення, `Tab` / `Shift+Tab` — перехід між полями.

## ▶️ Запуск

```
1. Завантажити або клонувати репозиторій
2. Відкрити KartingClubApp.sln у Visual Studio
3. Запустити (F5) або зібрати (Ctrl+Shift+B) і запустити KartingClubApp.exe
```

Файли `racers.json` і `sessions.json` створюються автоматично у папці з виконуваним файлом при першому збереженні.
