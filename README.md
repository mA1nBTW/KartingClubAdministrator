# Адміністратор картинг-клубу 🏎️

**Курсова робота з дисципліни «Об'єктно-орієнтоване програмування»**

Програма мовою C# для автоматизації роботи адміністратора картинг-клубу. Проєкт реалізує класичні принципи ООП, забезпечує надійне зберігання даних та інтуїтивне управління інформацією про клієнтів (гонщиків) і спортивні події (заїзди).

## ⚙️ Основний функціонал

* **Управління гонщиками (CRUD):** Реєстрація нових клієнтів, редагування профілів, безпечне видалення (із захистом від видалення учасників запланованих заїздів).
* **Управління заїздами (CRUD):** Створення заїздів, реєстрація учасників, фіксація часу кола (результатів).
* **Розумна статистика:** Автоматичне оновлення показників гонщика (кількість заїздів, перемог, найкращий час кола) після зміни статусу заїзду на «Завершено».
* **Пошук та фільтрація:** Швидкий пошук інформації за визначеними критеріями.
* **Збереження даних:** Безпечна серіалізація та десеріалізація стану системи у форматі JSON без дублювання об'єктів у пам'яті.
* **Стійкість (Валідація):** Сувора перевірка вхідних даних (вік, формат телефону, унікальність ліцензії).

## 🏗️ Архітектура та принципи ООП

Проєкт побудований з суворим дотриманням принципів SOLID (зокрема Single Responsibility Principle) та використовує чітко визначені типи взаємовідносин між класами:

1. **Клас `Racer` (Гонщик):** Основна модель даних клієнта (ID, ПІБ, вік, категорія, ліцензія, телефон).
2. **Клас `RacerStatistics` (Статистика):** Виділений клас для зберігання спортивних досягнень (кількість перегонів, перемог, найкращий час `TimeSpan`). Зв'язок із класом `Racer` реалізовано через **композицію** (статистика є невіддільною частиною профілю гонщика і видаляється разом з ним).
3. **Клас `Session` (Заїзд):** Модель події на трасі. Для зв'язку з гонщиками використовується **слабка асоціація** — клас зберігає лише унікальні ідентифікатори учасників (`List<int>`), а не посилання на самі об'єкти. Це гарантує відсутність циклічних посилань та дублювання даних під час збереження у JSON.
4. **Клас `KartingClubManager` (Менеджер):** Реалізує структурний патерн **«Фасад»**. Здійснює **агрегацію** головних колекцій (`List<Racer>`, `List<Session>`), інкапсулює всю бізнес-логіку (наприклад, автоматичний перерахунок статистики після гонки) та взаємодіє з файловою системою.
